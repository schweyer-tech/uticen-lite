# Learnings Index — controlflow-sdk

Durable, hard-won engineering rules harvested from this repo. Each row links to a file with full
context, the enforceable rule, and a reference. Read the relevant file by ID before doing work in
that area; treat active rules as binding. Newest entries are on top.

> ⚑ **[0001](0001-stay-compatible-with-the-controlflow-app.md) is the cardinal rule** — read it before
> any change that touches the bundle. The SDK exists to feed the ControlFlow app; staying
> bundle-compatible (`contract/bundle.schema.json`) is the one compatibility surface that must never
> silently drift.

| [0020](0020-keep-violation-details-json-native-sanitize-at-from-raw.md) | 2026-06-22 | data-integrity | Keep violation `details` JSON-native by sanitizing once at `Violation.from_raw` (the single funnel for the rule, pipeline, and Python paths) — a no-code condition on a `date`/`number`/`boolean` column emits a generic `{col: row[col]}` that leaks a pandas `Timestamp`/numpy scalar/`NaT` into `details` and breaks `json.dumps` on run-persist; duck-type the coercion (Timestamp→isoformat, NaT/NaN→None **before** isoformat, numpy→`.item()`) so `model/` stays pandas-free, and note `details` never enter the bundle so `schema_version` is untouched. | active |
| [0019](0019-put-the-worktree-on-pythonpath-when-verifying-its-code-with-an-adhoc-script.md) | 2026-06-22 | process | When verifying a worktree's changes by **running code** (a render/screenshot/repro script, not `pytest`/`python -m` from the worktree), put the worktree root on the path ahead of site-packages — `PYTHONPATH="$PWD" python /abs/script.py` or run from inside the worktree — because `python /abs/script.py` prepends the *script's* dir (not cwd) to `sys.path`, so `import <pkg>` resolves to the **editable-installed main checkout**, silently rendering the OLD code; assert a string the change introduced is present before trusting the output. | active |
