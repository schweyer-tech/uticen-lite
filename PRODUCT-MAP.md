# Product Map — controlflow-sdk

> Present-state inventory of what the SDK **ships now**. Planned/unbuilt work lives in GitHub issues,
> not here. Update a row when a surface ships or changes; remove a row when a surface is retired.
> Last updated: 2026-06-21.

| Surface | Type | What it does |
| --- | --- | --- |
| Distribution / install | docs + release CI | Corporate-ready install paths — pipx, internal index (`--index-url`/`--extra-index-url`), and pinned-wheel/air-gapped — documented in `docs/INSTALL.md`. Wheel is pure-Python `py3-none-any`, ships the web assets + bundled Northwind demo (verified by `tests/plane/test_wheel_build.py`). A manual, token-gated `release.yml` (`workflow_dispatch`, `PYPI_API_TOKEN`) builds + `twine check`s the dist; **not yet published to PyPI**. |
| `controlplane` | command / web app | Launches the local SQLite-backed FastAPI + HTMX control plane (localhost only, default `:8765`); the primary authoring surface. `pip install 'controlflow-sdk[plane]'` → `controlplane --project <dir>` (or the equivalent `python -m controlflow_sdk.plane`). |
| Control plane — First-run setup | view | Shown when an engagement has no name yet: name the engagement + pick a framework, or one-click **Load the Northwind demo** (bundled in the wheel, copied into the engagement so it runs immediately). |
| Control plane — Dashboard | view | Lists controls + last-run status; entry point to author, run, and export. Engagement name renders in the header. |
| Control plane — Source manager | view | List engagement sources (friendly title + code id); **Add source** opens a dedicated upload page that infers columns, then lands on the source editor. Persisted to `controlplane.db`. |
| Control plane — Source editor (Definition / Data / History tabs) | view | Three server-rendered tabs: **Definition** (title, description, key columns, column mapping); **Data** (current-file record count, read-only paged row preview, a **Mapping check** callout flagging columns that coerced to all-empty for a non-text declared type, per-file **as-of date**, and **Refresh data** → review column diff → confirm, archiving the prior file); **History** (every uploaded file version with as-of date, record count, upload time, current badge). Per-file data lineage in a store-only `source_files` table; `sources.extract_date` mirrors the current file's as-of (bundle path unchanged). |
| Control plane — Control editor (Definition / Logic / History tabs) | view | **Definition** tab is metadata-only: title, objective, narrative, framework refs, failure thresholds, and source binding — a control's **test logic** is authored in the **Logic** tab (next row), never here. **History** lists prior runs (see Run history). |
| Control plane — Logic (Builder / Flowchart / Python tabs) | view | One place to author a control's test logic (`/controls/{id}/logic/...`) — three server-rendered sub-tabs over a single node graph. **Builder**: every control is a node graph (Import · Filter · Join exists/not-exists/inner/left · Test · Custom Python) — a simple rule is `Import→Test`; rule_spec/empty controls open as a derived/scaffold graph (editable, incl. cross-source conditions); raw-Python controls show a notice. **Live row-counts** per node + a **Draft with AI** affordance. **Flowchart**: a read-only **multi-lane SVG** (parallel branches converge at joins). **Python**: the read-only **generated-Python** glass-box + a one-way **Convert to Python test**, or the editable **Python escape hatch** (CodeMirror) for hand-written controls. The graph is store-only and **compiles** to a `rule_spec` (pure single-source) or a `test(pop, sources)` string at run/build — bundle shape unchanged. Custom Python nodes are file/source-starved (allowlist AST lint + lexical starvation + hard export gate). |
| Control plane — AI-assisted draft | view | **Draft with AI** in the Logic ▸ Builder + a `/settings/ai` provider/model picker. Provider-agnostic (Anthropic / OpenAI / local Ollama) behind the optional `[ai]` extra, lazy-imported; keys from env vars only; **offline by default**. Drafts a `rule_spec` from the objective + bound-source schema + a data sample and **applies it into the Builder's Test node** (editable in place; persisted only on Save), re-validated (`parse_rule_spec` + run-on-sample) before it can be accepted. |
| Control plane — Run view | view | Per-run results (totals + violations table) with the **rendered workpaper embedded**. |
| Control plane — Run history | view | Per-control run history (`/controls/{id}/history`, linked from the dashboard + control editor): every run with pass-rate/exception counts, each row linking to its run view, plus an inline-SVG **trend chart** (pass-rate line + per-run exception bars with labeled axes). Read-only over stored runs. |
| Control plane — Export | view | Produces the import bundle (zip) for the ControlFlow app; validates against the contract. |
| Control plane — Settings | view | A **Settings** hub (`/settings`) linking engagement settings — **rename** the current engagement (single-engagement by design) — and the AI provider picker (`/settings/ai`, which names the exact env var per provider). |
| `cflow import <yaml> [--into <dir>]` | CLI | One-time import of a YAML project into `controlplane.db`. |
| `cflow run [dir] [--control] [--at]` | CLI | Run controls over the store, full-population; write workpapers + evidence; persist runs. |
| `cflow build [dir] [--out] [--at]` | CLI | Assemble + write the import bundle from the store. |
| `cflow validate [dir]` | CLI | Light DB integrity check (deprecated stub; prefer the web app). |
| `def test(pop[, sources])` | authoring API | Python full-population test; optional 2nd arg = `{source_id: Population}` for cross-source joins. |
| `rule_spec` + no-code rule builder | authoring | Declarative rule (14 operators incl. a cross-source **exists_in/not_exists_in** primitive, AND/OR logic, severity, description template, item-key) → violations; cross-source rules render to plain-Python `test_code` so the bundle shape is unchanged. Same execution contract as Python. |
| Bundle + `contract/bundle.schema.json` | integration contract | The import shape the ControlFlow app consumes 1:1 (definitions + run provenance; no raw population). |
| Workpaper renderer (HTML + Markdown) | output | Self-contained, audit-grade workpaper: Results, Objective & Scope, Control, Data Sources, Procedures, Exceptions, Conclusion. |
| `examples/northwind-trading/` | demo / template / fixture | An 8-control engagement (financial close / IT access / procurement, NIST 800-53 mapped) authored across a **mix of modes** — no-code rules, visual pipelines, and Python — to demonstrate the authoring ladder; runnable demo, cold-user template, and end-to-end CI fixture. |
